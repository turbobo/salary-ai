## 快速开始

1. **打开工具** — 在浏览器中访问应用地址
2. **选择城市** — 在「城市与基础设置」中选择你的工作城市，系统会自动加载对应的社保公积金比例
3. **填写月薪** — 在「全局参数设置」中输入你的税前月薪
4. **查看结果** — 页面下方会实时显示全年 12 个月的薪资明细和汇总数据
5. **（可选）登录** — 点击右上角「登录」，可将数据保存到云端
6. **（可选）使用 AI** — 展开「AI 智能助手」面板，体验智能输入和分析功能

> 无需注册即可使用所有核心功能。未登录状态下数据自动保存在浏览器本地。

## 全局参数设置

全局参数将作为所有月份的默认值。你可以在此统一设置：

- **固定月薪（税前）** — 每月税前工资金额
- **社保基数** — 社保缴费基数，留空则默认等于月薪
- **养老/医疗/失业保险比例** — 填写百分比数值，如 `8` 表示 8%
- **公积金基数** — 公积金缴费基数，留空则默认等于月薪
- **公积金比例** — 个人缴存比例

点击 **「一键应用到全部月份」** 按钮，会清除所有月份的单独设置，全部使用全局值。

## 月度薪资设置

### 月份选择

页面顶部有 1-12 月的标签，点击可启用或禁用该月参与计算。例如：年中入职只需勾选入职后的月份。

### 月度卡片

每个月份有独立的卡片，可设置：

| 字段 | 说明 |
|------|------|
| 月薪(税前) | 该月税前工资，留空使用全局值 |
| 补贴(计税) | 该月需要计税的补贴，如交通补贴、餐补等 |
| 奖金(免税) | 该月免税奖金，如某些一次性发放的津贴 |
| 社保基数 | 该月社保缴费基数，留空使用全局值 |
| 公积金基数 | 该月公积金缴费基数，留空使用全局值 |
| 公积金比例 | 该月公积金比例，留空使用全局值 |

点击月份卡片标题栏可折叠/展开详细设置。

## 年终奖设置

### 年终奖金额

输入年终奖总额（元）。

### 发放月份

选择年终奖计入哪个月份（影响合并计税时的累计税额计算）。

### 计税方式

| 方式 | 说明 | 适用场景 |
|------|------|---------|
| **单独计税** | 年终奖 / 12，按月度税率表单独计算税额 | 年终奖金额较大、月薪较高时通常更优 |
| **合并计税** | 年终奖并入发放月份的工资，按累计预扣预缴法计算 | 年收入较低、年终奖较少时可能更优 |

> 提示：可以在 AI 税务优化功能中自动对比两种方式，查看哪种交税更少。

## AI 智能助手

点击页面中的 **「AI 智能助手」** 面板标题可展开/折叠。

### AI 模式说明

- **演示模式（默认）**：无需 API Key，使用内置算法生成参考数据
- **API 模式**：输入通义千问 DashScope API Key，获得 AI 实时分析

API Key 设置方法：在页面顶部「DashScope API Key」输入框中粘贴你的 Key，点击保存。Key 仅存储在浏览器本地，AI 请求通过服务端代理转发，不会暴露 Key。

### 1. 智能输入

用自然语言描述你的薪资情况，AI 会自动提取信息并填入表单。

**示例输入：**
```
我在上海工作，月薪3万，公积金12%，年终奖6万，有房贷利息扣除1000/月
```
```
北京工作，月薪25000，社保基数30000，年终奖3个月工资，赡养老人扣除2000
```

**支持识别的信息：** 城市、月薪、社保基数、公积金基数/比例、年终奖、专项附加扣除

AI 解析完成后会显示识别结果，确认无误后点击 **「确认应用」** 即可自动填入。

### 2. 薪资竞争力分析

基于你的薪资和所在城市，给出：

- **竞争力评分**（1-100 分）
- **城市平均薪资对比**
- **所处百分位排名**
- **综合有效税率**
- **个性化优化建议**

### 3. 税务优化分析

分为两部分：

**本地精确计算** — 自动对比单独计税与合并计税的总税额差异，显示哪种方式更省税，以及可节省的金额。可一键切换到更优的计税方式。

**AI 建议** — 提供专项附加扣除优化建议、个人养老金账户等节税策略。

### 4. 职业规划建议

根据你的薪资水平和城市信息，提供：

- 当前薪资定位分析
- 短/中/长期薪资增长路径
- 技能提升方向建议
- 个人财务规划建议

## 常见问题 FAQ

**Q: 数据安全吗？会不会泄露？**
A: 未登录用户数据仅保存在浏览器 localStorage 中。登录用户数据保存在 Supabase 数据库中，通过行级安全策略（RLS）确保只有你自己能访问。AI API Key 仅存储在浏览器本地，AI 请求通过服务端代理转发。

**Q: 不注册能用吗？**
A: 完全可以。所有核心功能（薪资计算、AI 分析、CSV 导出）均不需要注册登录。注册仅用于云端保存和历史记录。

**Q: 换一台电脑/浏览器，数据还在吗？**
A: 未登录用户的数据保存在当前浏览器本地，换设备后不可用（可导出 CSV 备份）。已登录用户可在任何设备上登录后从历史记录中加载数据。

**Q: 不输入 API Key 能用 AI 功能吗？**
A: 可以。AI 功能在未配置 API Key 时以「演示模式」运行，使用内置算法提供参考数据。

**Q: 支持哪些 AI 模型？**
A: 使用阿里云通义千问 `qwen-plus` 模型，通过 DashScope API 调用。

**Q: 「跳档」是什么意思？**
A: 中国个税采用累进税率，当你的累计应纳税所得额超过某一档的上限时，超出部分按更高税率计算，这就是「跳档」。跳档月份的个税会明显增加。

**Q: 社保基数和月薪有什么关系？**
A: 社保基数是计算社保缴费的基准金额，通常等于你上一年的月平均工资。每个城市有最低和最高限制，系统会自动在范围内夹紧。如果不单独设置，默认使用当月工资作为基数。

**Q: 年终奖应该选单独计税还是合并计税？**
A: 取决于你的总收入水平。可以使用 AI 税务优化功能自动对比两种方式的税额差异，选择税额更低的方式。

## 核心计算逻辑

核心计算引擎位于 `src/lib/calculation/`，从原始 `index.html` 中零改动提取，添加了 TypeScript 类型。

### 累计预扣预缴法

`calculateMonthlyResults()` 函数（`salary.ts`）实现了中国个税累计预扣预缴算法：

```
本月应预扣预缴税额 = (累计预扣预缴应纳税所得额 × 税率 - 速算扣除数) - 累计已预扣预缴税额

累计预扣预缴应纳税所得额 = 累计收入 - 累计免税收入 - 累计减除费用 - 累计专项扣除 - 累计专项附加扣除
```

### 个税税率表（综合所得）

| 级数 | 累计应纳税所得额 | 税率 | 速算扣除数 |
|------|----------------|------|-----------|
| 1 | 0 - 36,000 | 3% | 0 |
| 2 | 36,000 - 144,000 | 10% | 2,520 |
| 3 | 144,000 - 300,000 | 20% | 16,920 |
| 4 | 300,000 - 420,000 | 25% | 31,920 |
| 5 | 420,000 - 660,000 | 30% | 52,920 |
| 6 | 660,000 - 960,000 | 35% | 85,920 |
| 7 | 960,000 以上 | 45% | 181,920 |

### 年终奖单独计税税率表

年终奖 / 12 后按以下月度税率表计算：

| 月均金额 | 税率 | 速算扣除数 |
|---------|------|-----------|
| 0 - 3,000 | 3% | 0 |
| 3,000 - 12,000 | 10% | 210 |
| 12,000 - 25,000 | 20% | 1,410 |
| 25,000 - 35,000 | 25% | 2,660 |
| 35,000 - 55,000 | 30% | 4,410 |
| 55,000 - 80,000 | 35% | 7,160 |
| 80,000 以上 | 45% | 15,160 |

### 社保公积金计算

```
社保缴费 = clamp(社保基数, 城市下限, 城市上限) × 个人缴纳比例
公积金缴费 = clamp(公积金基数, 城市下限, 城市上限) × 个人缴存比例
```

基数默认等于当月工资，系统会自动按城市规定的上下限进行夹紧。

## AI / DashScope API 接入

### 架构

AI 调用采用服务端代理模式，保护 API Key 安全：

```
浏览器 → POST /api/ai → Next.js API Route → DashScope API
                ↑
        可选：用户自己的 API Key
        或：服务端环境变量 DASHSCOPE_API_KEY
```

### API 配置

| 配置项 | 值 |
|-------|-----|
| API 地址 | `https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions` |
| 模型 | `qwen-plus` |
| 调用方式 | OpenAI 兼容模式（HTTP POST，Bearer Token） |
| 参数 | `temperature: 0.7`, `max_tokens: 2000` |

### 获取 API Key

1. 访问 [阿里云 DashScope 控制台](https://dashscope.console.aliyun.com/)
2. 注册/登录后创建 API Key
3. 方式一：填入 `.env.local` 的 `DASHSCOPE_API_KEY`（所有用户共享）
4. 方式二：用户在页面顶部输入自己的 Key（仅存储在该用户浏览器本地）

## Supabase 数据库

### 数据表

单表 `calculations`，使用 JSONB 存储灵活的月度数据：

```sql
CREATE TABLE calculations (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id          UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  name             TEXT,                        -- 用户标签，如 "上海 - 2024/6/4"
  city             TEXT NOT NULL,
  global_params    JSONB NOT NULL,              -- 全局薪资参数
  monthly_data     JSONB NOT NULL,              -- 12个月的逐月数据
  enabled_months   JSONB NOT NULL,              -- 启用的月份
  annual_bonus     NUMERIC DEFAULT 0,
  bonus_month      INTEGER DEFAULT 1,
  bonus_tax_method TEXT DEFAULT 'separate',
  special_deduction NUMERIC DEFAULT 0,
  total_gross      NUMERIC,                     -- 汇总数据（用于列表展示）
  total_net        NUMERIC,
  total_tax        NUMERIC,
  created_at       TIMESTAMPTZ DEFAULT now(),
  updated_at       TIMESTAMPTZ DEFAULT now()
);
```

### 行级安全策略 (RLS)

用户只能操作自己的数据：

```sql
ALTER TABLE calculations ENABLE ROW LEVEL SECURITY;
CREATE POLICY "own_data" ON calculations FOR ALL USING (auth.uid() = user_id);
```

### 认证配置

在 Supabase 控制台中：
1. **Authentication > Providers > Email** — 启用邮箱密码注册
2. **Authentication > Providers > GitHub** — 配置 GitHub OAuth App 的 Client ID 和 Secret
3. **Authentication > URL Configuration** — 设置 Site URL 和 Redirect URLs

## 本地部署与二次开发

### 环境要求

- Node.js 18+
- npm

### 本地运行

```bash
npm install          # 安装依赖
npm run dev          # 启动开发服务器 (http://localhost:3000)
```

### 生产构建

```bash
npm run build        # 构建
npm start            # 启动生产服务器
```

### 二次开发要点

**修改城市数据**：编辑 `src/lib/calculation/constants.ts` 中的 `CITY_PRESETS`，添加新城市或更新年度基数。

**更换 AI 模型**：修改 `src/app/api/ai/route.ts` 中的 `DASHSCOPE_API_URL` 和 `AI_MODEL`。只要 API 兼容 OpenAI Chat Completions 格式即可。

**自定义样式**：修改 `src/app/globals.css` 中的 CSS 变量（颜色主题）或各组件中的 Tailwind 类名。

**扩展数据库**：在 `supabase/migrations/` 中添加新的迁移文件，保持 RLS 策略。
